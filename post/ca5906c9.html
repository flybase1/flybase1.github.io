网关的作用

发表于2023-07-26|更新于2023-07-29|网关
|阅读量:

  1. 路由
  2. 鉴权
  3. 缓存
  4. 跨域
  5. 流量染色
  6. 访问控制
  7. 统一业务处理
  8. 负载均衡
  9. 发布控制
  10. 统一日志
  11. 统一文档
  12. 业务保护
    1. 限制请求
    2. 信息脱敏
    3. 降级（熔断）
    4. 限流
    5. 超时时间

2. 路由

起到转发作用，将请求转发到对应的接口、服务器、集群

3. 负载均衡

在路由的基础上，可以随机转发到某一台机器。

4. 鉴权

判断用户是否有权限进行操作，无论访问什么接口，都能够统一判断。

5. 统一处理跨域

统一处理跨域，不用再某个项目里面单独写。

6. 统一业务处理

统一处理业务，把每个项目都要用到的通用逻辑都放到上层里面去统一处理，比如调用次数

7. 访问控制

黑白名单，比如限制DDOs IP

8. 发布控制

局部试错，比如添加了新接口，给老接口分配80%流量，新接口20%，然后逐步过渡到新接口
Weight route

9. 流量染色

区分用户的来源，判断用户是否从网关来的
给请求添加一些标识，一般是设置请求头
AddRequest header

10. 接口保护

  1. 限制请求
  2. 信息脱敏
  3. 降级（熔断）
  4. 限流
  5. 超时时间

11. 统一日志

统一的请求响应信息记录

12. 统一文档

将下游项目的文档进行聚合，在同一个界面进行查看

13. 网关分类

  1. 业务网关（微服务网关）：负载均衡
  2. 全局网关（层级更高）：将一些业务逻辑，将一些请求转发到不同的接口/业务/服务器

14. 实现

  1. nginx、kong（API服务）
  2. Spring cloud gateway
